# fo-u15.3.3 — Cluster Render: collapsed + expand-on-demand

date: 2026-05-17
bead: fo-u15.3.3
parent epic: fo-u15.3 (failure clustering)
predecessors: fo-u15.3.1 (heuristic), fo-u15.3.2 (wire into Report)

## Goal

Render `Report.Clusters` in human and llm modes so that 40 tests failing from one root cause occupy proportional screen/token space, not 40x the space of one failure.

## Reader contract

### human

Clusters collapse by default. Each cluster renders as:

```
▸ pkg/store.(*DB).Get · 12 tests · --expand=cluster-a3f2c1
  • TestGet_NotFound — assertion: got nil, want ErrMissing
```

- Header line: signature (cluster's anchor frame or assertion text), member count, expand hint. The cluster ID lives only in the hint — no redundant standalone token. `▸` is a static marker, not a clickable disclosure (fo is one-shot stdin→stdout).
- One representative member: `Report.Tests` source order — the first `TestResult` whose `ClusterID` matches. Renders identically to a singleton test line.
- Members beyond the representative are hidden.

When `--expand=cluster-a3f2c1` or `--expand=all` is set, that cluster renders all members (header + every member as a test line, no rep distinction).

### llm

Clusters always render full — no collapse, no flag effect. Two block shapes, picked per cluster by content:

**Shape A — shared output dedupe** (all members have byte-identical failure text):

```
cluster cluster-a3f2c1 · pkg/store.(*DB).Get · 3 tests
  shared: got nil, want ErrMissing
  members: TestGet_NotFound, TestGet_NotFound_TX, TestGet_AfterDelete
```

**Shape B — header + per-member** (members diverge on output, share frame only):

```
cluster cluster-a3f2c1 · pkg/store.(*DB).Get · 3 tests
  TestGet_NotFound: got nil, want ErrMissing
  TestGet_NotFound_TX: got 0 rows, want 1
  TestGet_AfterDelete: panic: nil map
```

No column padding in Shape B — the llm reader pays per token; visual alignment is a human concern.

Shape detection: **byte equality** over each member's raw `TestResult.Output`. Identical → Shape A; any divergence → Shape B. Deliberately conservative: the clusterer's normalizer (`pkg/cluster/normalize`) strips numbers/paths/addrs and would collapse genuinely-divergent failures behind one `shared:` line. Display equality must be honest.

### theme

Cluster header uses `theme.Heading` for the signature + count (groups it visually as a section) and `theme.Muted` for the `--expand=…` hint (de-emphasizes the affordance — it's metadata, not content). Member lines under an expanded cluster use the same theme tokens as singleton test lines (`Pass`/`Fail`/`Skip`/etc. per outcome). Under `Mono`, both `Heading` and `Muted` degrade gracefully (bold + faint, no color) so cluster headers stay distinguishable from member lines in mono mode.

### cluster ID stability

IDs are `cluster-<hex>` where the hex is a sha256 prefix of the cluster signature (`pkg/cluster/id.go: makeClusterID`). Stable across runs as long as the signature is stable — fixing one cluster's tests does not renumber the others. The only instability vector is the `-N` collision suffix appended by `disambiguate` when two distinct signatures happen to share the truncated hex prefix; in that case the relative order of clashes determines which one gets the bare ID and which gets `-2`, etc. For the current `IDHexLen` this is rare in practice but not impossible. Documented here so a CI script that pins `--expand=cluster-X` understands the failure mode.

### cluster ordering

Render order follows `Report.Clusters` order, which C.2 produces sorted by member count descending, ID ascending as tiebreaker (`pkg/cluster/cluster.go:162`). This is the right rendering order for token-saving — the biggest dedupe wins appear at the top of the output. If C.2's ordering changes, this view degrades silently (small clusters first, large clusters buried), so any C.2 change that touches ordering should re-check the goldens here.

### singletons

Tests with empty `ClusterID` render unchanged through the existing `pickview`
pipeline. Cluster representatives are not also emitted as singletons —
partition is exclusive.

## Flag: `--expand`

- Repeatable: `--expand=cluster-a3f2c1 --expand=cluster-b7e009`.
- Value `all`: reveal every cluster.
- Unknown cluster ID: stderr warning `fo: --expand=cluster-deadbeef not found`. Exit code unchanged (0/1 follows findings/test outcome as today).
- Mixed known + unknown in one invocation: known IDs expand normally, unknown IDs each emit their own stderr warning. No fail-fast.
- In llm mode: silent no-op. llm is always full; agents may pass the flag uniformly without inspecting render mode.
- In json mode: silent no-op (json passes `Clusters` through as a structured field).

## Data flow

```
Report.Tests + Report.Clusters
   │
   ├─ pickview / view layer:
   │     partition Tests into:
   │       clustered: map[ClusterID][]TestResult  (in source order)
   │       singletons: []TestResult
   │
   ├─ human renderer (scene_human):
   │     for each Cluster in Report.Clusters order:
   │       header line
   │       if expand-set hits this ID OR expand=all:
   │         all members as test lines
   │       else:
   │         representative only
   │     then singletons unchanged
   │
   └─ llm renderer (scene_llm):
         for each Cluster:
           dedupe-detect on shared output string
           emit Shape A or Shape B
         then singletons unchanged
```

## Components

### new — `pkg/view/cluster.go`

Pure-formatting helpers (no render-mode policy, no flag-state):

- `partition(tests []TestResult, clusters []Cluster) (map[string][]TestResult, []TestResult)` — split clustered from singletons by `ClusterID`.
- `clusterHeader(c Cluster, k int) string` — `▸ <signature> · K tests · --expand=<id>`.
- `sharedOutput(members []TestResult) (string, bool)` — byte-equality check over members' raw `Output`. All match → returns the (identical) `Output` and `true`. Any divergence → `("", false)`. No normalization on the render path.
- `expandSet` type — `{ ids map[string]struct{}; all bool }`. `func (e expandSet) wants(id string) bool`.

### modified — `pkg/view/scene_human.go`

Hook cluster rendering ahead of the existing test-line emission. Expanded clusters emit member lines using the same formatter as singletons.

### modified — `pkg/view/scene_llm.go`

Same hook. No flag consultation — llm always full.

### modified — `cmd/fo/main.go`

`--expand` flag: `StringSliceVar` (or equivalent). Parse into `expandSet`:
- `all` → set `all=true`.
- `cluster-*` → add to `ids`.
- Validate against `Report.Clusters` IDs after parse; unknown → stderr warning.
- Thread `expandSet` into existing render-options struct (alongside format, theme, etc.).

## Testing

### unit (`pkg/view/cluster_test.go`)

- partition: clustered + singletons split correctly; representative is source-order first match; multi-cluster partition.
- clusterHeader: known signature; long signature truncation if format already truncates elsewhere.
- sharedOutput: all-identical → `(text, true)`; one diverges → `("", false)`; empty members → `("", false)`.
- expandSet.wants: id match, all-overrides, miss.

### golden (`pkg/view/scene_human_test.go`, `scene_llm_test.go`)

Cases:
1. **human collapsed** — 1 cluster (3 members) + 2 singletons → 1 header + 1 rep + 2 singletons.
2. **human expanded by id** — same input with `--expand=cluster-1` → 1 header + 3 members + 2 singletons.
3. **human expanded all** — same input with `--expand=all` → 1 header + 3 members + 2 singletons.
4. **human unknown id** — `--expand=cluster-999`, no matching cluster → collapsed output unchanged, stderr warning captured.
5. **llm Shape A** — 3 members, identical assertion text → header + `shared:` + `members:`.
6. **llm Shape B** — 3 members, divergent text → header + per-member lines.
7. **llm with flag** — `--expand=all` passed; output identical to (5) or (6) (flag is no-op).
8. **mixed clusters + singletons** — 2 clusters of different shapes + 1 singleton in both human and llm.
9. **human mixed known + unknown expand** — `--expand=cluster-<known> --expand=cluster-deadbeef`: known cluster expands, unknown emits stderr warning, render proceeds.
10. **theme parity** — Color and Mono produce structurally equivalent output (same lines, same lengths after ANSI strip); cluster header is distinguishable from member lines in both.

### cmd/fo integration

- Flag parses comma-and-repeat forms.
- Unknown ID emits the expected stderr line.
- Exit codes match pre-flag behavior (verifies flag is observation, not gate).

## Acceptance

- Cluster of 12 tests in human mode renders as 2 lines (header + rep), down from 12.
- Same cluster in llm mode renders as 4 lines (Shape A: header + shared + members + ∅) or 1+12 (Shape B), down from 12 independent failure blocks.
- `--expand=cluster-X` reveals members of cluster X in human mode; other clusters stay collapsed.
- `--expand=all` reveals every cluster in human mode.
- `--expand=cluster-999` emits stderr warning and proceeds.
- llm output is identical with or without `--expand`.
- Singletons render unchanged; no test appears in both a cluster and the singletons list.
- All existing scene_human / scene_llm goldens still pass (cluster path is additive).
